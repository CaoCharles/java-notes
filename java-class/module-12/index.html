    Module 12: 例外處理、日誌與除錯 (Exception Handling, Logging, and Debugging)¶

    Java SE 21 程式語言設計

    學習目標¶

    完成本章節後，您將能夠： - 區分 Checked Exception, Unchecked Exception 與 Error - 使用 try-catch-finally 處理例外 - 使用 try-with-resources 自動關閉資源 - 建立與拋出自訂例外 - 使用 Java Logging API 記錄日誌 - 使用 Assertions 進行程式碼驗證

    1. 例外 (Exceptions) 概觀¶

    例外是程式執行期間發生的異常事件，會中斷正常的指令流程。Java 的例外階層如下：

    • Throwable (所有錯誤與例外的父類別)
      • Error: 嚴重的系統錯誤，通常無法復原 (e.g., OutOfMemoryError, StackOverflowError)。
      • Exception: 程式可以處理的異常狀況。
        • RuntimeException: 程式邏輯錯誤 (e.g., NullPointerException)。

    Checked vs Unchecked Exceptions¶

    特性 Checked Exceptions (受檢例外) Unchecked Exceptions (非受檢例外)
    繼承自 Exception (但不含 RuntimeException) RuntimeException
    編譯器檢查 是 (必須處理) 否 (選擇性處理)
    代表意義 外部環境導致的錯誤 (如檔案遺失、網路斷線) 程式邏輯錯誤 (Bug)
    處理方式 try-catch 或 throws 修正程式碼邏輯
    範例 IOException, SQLException NullPointerException, ArithmeticException

    BufferedReader in = null;
try {
    // 可能拋出例外的程式碼
    in = new BufferedReader(new FileReader("data.txt"));
    String line = in.readLine();
} catch (FileNotFoundException e) {
    // 處理特定例外
    System.err.println("檔案找不到: " + e.getMessage());
} catch (IOException e) {
    // 處理其他 IO 例外
    System.err.println("讀取錯誤: " + e.getMessage());
} finally {
    // 無論是否發生例外，都會執行的區塊 (通常用於釋放資源)
    if (in != null) {
        try {
            in.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

    Java 7 引入，針對實作 AutoCloseable 介面的資源 (如 IO Stream, JDBC Connection)，結束時會自動呼叫 close()。

     PrintWriter out = new PrintWriter(new FileWriter("output.txt"))) {

    String line = in.readLine();
    out.println(line);

} catch (IOException e) {
    System.err.println("IO Error: " + e.getMessage());
}

        throw new FileNotFoundException("File not found: " + filename); // Checked

    4. Java Logging API¶

    使用 java.util.logging 記錄程式執行資訊，比 System.out.println 更具彈性與效能。

    • Logger: 記錄器。
    • Level: 嚴重程度 (SEVERE > WARNING > INFO > CONFIG > FINE > FINER > FINEST)。
    • Handler: 輸出目的地 (Console, File, Socket)。

    import java.util.logging.*;

public class LoggingDemo {
    private static final Logger logger = Logger.getLogger(LoggingDemo.class.getName());

    public static void main(String[] args) {
        logger.info("Application started");

        try {
            int result = 10 / 0;
        } catch (ArithmeticException e) {
            // 記錄例外堆疊
            logger.log(Level.SEVERE, "Calculation error", e);
        }
    }
}

    5. Assertions (斷言)¶

    用於開發與測試階段，驗證程式的假設是否成立。 - 語法: assert condition : message; - 執行時需加上 -ea (Enable Assertions) 參數才會生效。

    Java
    1

    public void setDiscount(double discount) {
    // 假設折扣必須在 0 到 1 之間
    assert discount >= 0 && discount <= 1 : "Discount must be between 0 and 1";

    this.discount = discount;
}
    注意: 不要用 Assertions 檢查 public 方法的參數 (應使用 IllegalArgumentException)，因為 Assertions 在生產環境通常會被關閉。

    6. 最佳實務 (Best Practices)¶

    1. 不要吞掉例外 (Swallowing Exceptions): catch 區塊中至少要記錄 Log，不要留空。
    2. 使用具體的例外: 盡量捕捉特定的例外 (如 FileNotFoundException) 而非通用的 Exception。
    3. Fail Fast: 儘早檢查錯誤 (如參數驗證) 並拋出例外。
    4. 保留原始例外: 包裝例外時 (Wrapping Exception)，記得將原始例外傳入建構子，以免遺失 Stack Trace。
