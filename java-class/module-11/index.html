    Module 11: Java Stream API¶

    Java SE 21 程式語言設計

    學習目標¶

    完成本章節後，您將能夠： - 理解 Stream API 的核心概念 (Lazy Evaluation, Pipelining) - 使用 Stream 處理集合資料 (Filter, Map, Reduce) - 使用 Optional 類別處理可能為空的值 - 使用 Collectors 進行資料收集與分組 - 理解平行串流 (Parallel Streams) 的使用時機

    1. Stream API 簡介¶

    Stream 是一組支援函數式操作的元素序列，它不是資料結構，不儲存資料，而是從來源 (如 Collection, Array) 取得資料並進行運算。

    核心特性¶

    1. 不儲存資料: Stream 只是資料的搬運工。
    2. 不改變來源: 操作 Stream 不會修改原始資料來源，而是產生新的結果。
    3. Lazy Evaluation (延遲執行): 中介操作 (如 filter, map) 只有在終端操作 (如 collect, forEach) 被呼叫時才會真正執行。這能優化效能，例如只處理需要的元素。
    4. Internal Iteration (內部迭代): 由 Stream API 自動處理迭代，開發者只需關注「做什麼」而非「怎麼做」。

String[] array = {"X", "Y", "Z"};

Stream.of(1, 2, 3).forEach(System.out::println);

// 4. 產生無限串流 (需搭配 limit)
Stream.generate(Math::random).limit(5).forEach(System.out::println);

    2. Stream 操作¶

    Stream 操作分為兩類： - Intermediate Operations (中介操作): 回傳 Stream，可串接 (Chaining)。 - Terminal Operations (終端操作): 回傳結果或 void，觸發管線執行。

    2.1 常用中介操作¶

    • filter(Predicate<T>): 保留符合條件的元素。
    • map(Function<T, R>): 將元素轉換為另一種形式。
    • flatMap(Function<T, Stream<R>>): 將每個元素轉換為 Stream 後合併 (扁平化)。
    • distinct(): 去除重複元素 (依據 equals)。
    • sorted(): 排序 (自然排序或依 Comparator)。
    • limit(long n): 取前 n 個元素。
    • skip(long n): 跳過前 n 個元素。

    List<Product> menu = Product.getProducts();

menu.stream()
    .filter(p -> p.getPrice().compareTo(BigDecimal.TEN) > 0) // 價格 > 10
    .filter(p -> !p.getName().startsWith("A"))               // 名稱不以 A 開頭
    .map(Product::getName)                                   // 只取名稱
    .sorted()                                                // 排序
    .forEach(System.out::println);                           // 印出

    2.2 map vs flatMap¶

    • map: 一對一轉換 (例如：字串 -> 長度)。
    • flatMap: 一對多轉換 (例如：句子 -> 單字 Stream -> 合併為單一 Stream)。

    List<String> sentences = List.of("Hello World", "Java Stream");

// map: 回傳 List<String[]>
sentences.stream()
    .map(s -> s.split(" "))
    .forEach(arr -> System.out.println(Arrays.toString(arr)));

// flatMap: 回傳 List<String> (扁平化)
sentences.stream()
    .flatMap(s -> Arrays.stream(s.split(" ")))
    .forEach(System.out::println); // Hello, World, Java, Stream

    2.3 常用終端操作¶

    • forEach(Consumer<T>): 對每個元素執行動作。
    • count(): 回傳元素數量。
    • collect(Collector): 將結果收集為 List, Set, Map 等。
    • findFirst(), findAny(): 回傳 Optional。
    • anyMatch(Predicate), allMatch(Predicate), noneMatch(Predicate): 回傳 boolean。
    • reduce(BinaryOperator): 將所有元素歸納為單一結果。

    Optional<T> 是一個容器物件，可能包含或不包含非 null 值。用於避免 NullPointerException。

// 正確使用方式
result.ifPresent(p -> System.out.println(p.getPrice()));

// 取得值或預設值
Product p = result.orElse(new Product("Default", BigDecimal.ZERO));

// 拋出例外
Product p2 = result.orElseThrow(() -> new RuntimeException("Not found"));

    java.util.stream.Collectors 提供了豐富的收集功能。

    // 轉為 List
List<String> names = menu.stream().map(Product::getName).collect(Collectors.toList());

// 轉為 Set
Set<String> uniqueNames = menu.stream().map(Product::getName).collect(Collectors.toSet());

// 轉為 Map (Key: Name, Value: Price)
Map<String, BigDecimal> priceMap = menu.stream()
    .collect(Collectors.toMap(Product::getName, Product::getPrice, (p1, p2) -> p1)); // 處理 Key 重複

// 字串連接
String allNames = menu.stream().map(Product::getName).collect(Collectors.joining(", "));

    // 依價格分組
Map<BigDecimal, List<Product>> byPrice = menu.stream()
    .collect(Collectors.groupingBy(Product::getPrice));

// 依條件分區 (Partitioning) - Key 為 Boolean
Map<Boolean, List<Product>> cheapFood = menu.stream()
    .collect(Collectors.partitioningBy(p -> p.getPrice().intValue() < 5));

    5. 平行串流 (Parallel Streams)¶

    使用 parallelStream() 可以利用多核心 CPU 加速處理。

    適用時機: - 資料量很大。 - 每個元素的處理很耗時。 - 操作是無狀態 (Stateless) 且無相依性 (Non-interfering)。

    注意事項: - 平行處理有額外開銷 (分割與合併)，資料量小反而變慢。 - 避免在平行串流中使用會修改共享狀態的操作 (Thread-safety issue)。

    ParallelStream.java
    1
2
3
    double total = menu.parallelStream()
    .mapToDouble(p -> p.getPrice().doubleValue())
    .sum();

    6. 總結¶

    • Stream API 提供宣告式的方式處理資料集合。
    • 善用 filter, map, collect 組合出強大的資料處理管線。
    • Optional 幫助優雅地處理空值。
    • Collectors 提供強大的資料收集與統計功能。

    7. 練習建議¶

    1. 給定一個字串 List，找出所有長度大於 5 的字串，轉為大寫，並排序。
    2. 計算所有產品的總價格 (使用 reduce 或 mapToBigDecimal 搭配 sum)。
    3. 將產品依據類別 (Category) 分組，並計算每一組的平均價格。
