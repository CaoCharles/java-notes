    Module 10: 巢狀類別與 Lambda 表達式 (Nested Classes and Lambda Expressions)¶

    Java SE 21 程式語言設計

    學習目標¶

    完成本章節後，您將能夠： - 理解並使用四種巢狀類別：Static Nested Class, Member Inner Class, Local Inner Class, Anonymous Inner Class - 理解 Lambda 表達式的語法與用途 - 使用 Functional Interfaces (如 Predicate, Comparator) - 使用 Method References (方法參考) 簡化程式碼

    1. 巢狀類別 (Nested Classes)¶

    巢狀類別是定義在另一個類別內部的類別。使用巢狀類別的主要原因包括： - 邏輯分組: 如果一個類別只被另一個類別使用，將其嵌入可以使程式碼更緊湊。 - 封裝性: 巢狀類別可以存取外部類別的私有成員 (包括 private)。 - 可讀性與維護性: 將相關的類別放在一起。

    1.1 Static Nested Class (靜態巢狀類別)¶

    • 宣告為 static 的巢狀類別。
    • 行為類似於一般的頂層類別 (Top-level class)，只是被包裝在另一個類別中。
    • 只能存取外部類別的靜態成員。
    • 可以直接實例化，不需要外部類別的實例。

    public class Outer {
    // 靜態巢狀類別
    public static class StaticNested { 
        public void print() {
            System.out.println("Inside StaticNested class");
        }
    }
}

// 實例化方式
Outer.StaticNested nested = new Outer.StaticNested();
nested.print();

    1.2 Member Inner Class (成員內部類別)¶

    • 定義在類別中，但不是 static 的。
    • 屬於外部類別的**實例 (Instance)** 成員。
    • 可以存取外部類別的所有成員 (包含 private)。
    • 必須先建立外部類別的實例，才能建立成員內部類別的實例。

    1.3 Local Inner Class (區域內部類別)¶

    • 定義在**方法 (Method)** 內部的類別。
    • 範圍僅限於該方法內部。
    • 可以存取外部類別的成員，以及方法內的 final 或 effectively final (賦值後不再改變) 的區域變數。

    1.4 Anonymous Inner Class (匿名內部類別)¶

    • 沒有名稱的內部類別。
    • 通常用於**繼承一個類別**或**實作一個介面**，並同時建立其實例。
    • 常用於事件監聽器 (Event Listeners) 或回呼 (Callbacks)。

    }

    2. Lambda 表達式 (Lambda Expressions)¶

    Java 8 引入的重大特性，用於簡化 Functional Interface (只有一個抽象方法的介面) 的實作。

    2.1 語法¶

    Java
    1
    (parameters) -> { body }
    - 參數: 可以省略型別 (Type Inference)。如果只有一個參數，括號 () 可以省略。 - 箭頭: -> 分隔參數與主體。 - 主體: 如果只有一行程式碼，大括號 {} 和 return 關鍵字可以省略。

    2.2 範例：Comparator¶

    比較傳統匿名類別與 Lambda 的寫法。

    使用匿名類別:

    Java
    1
2

    Comparator<String> byLength = new Comparator<String>() {
    @Override
    public int compare(String s1, String s2) {
        return Integer.compare(s1.length(), s2.length());
    }
};

    使用 Lambda:

    ComparatorDemo01.java
    1
2
3
4

    // 完整寫法
Comparator<String> byLength = (String s1, String s2) -> {
    return Integer.compare(s1.length(), s2.length());
};

// 簡潔寫法
Comparator<String> byLengthSimple = (s1, s2) -> Integer.compare(s1.length(), s2.length());

    2.3 常用的 Functional Interfaces¶

    Java 在 java.util.function 套件中提供了許多標準介面：

    • Predicate<T>: 接收一個參數，回傳 boolean。用於測試條件。
      Java
      1
      Predicate<String> isEmpty = s -> s.isEmpty();
    • Consumer<T>: 接收一個參數，不回傳值 (void)。用於執行動作。
      Java
      1
      Consumer<String> printer = s -> System.out.println(s);
    • Function<T, R>: 接收參數 T，回傳結果 R。用於轉換。
      Java
      1
      Function<String, Integer> lengthFunc = s -> s.length();
    • Supplier<T>: 不接收參數，提供一個回傳值 T。用於產生資料。
      Java
      1
      Supplier<Double> random = () -> Math.random();

    3. Method References (方法參考)¶

    當 Lambda 表達式只是直接呼叫一個現有的方法時，可以使用方法參考來讓程式碼更簡潔。語法為 Class::method。

    類型 Lambda Method Reference
    Static Method args -> Class.staticMethod(args) Class::staticMethod
    Instance Method (特定物件) args -> obj.instanceMethod(args) obj::instanceMethod
    Instance Method (任意物件) (obj, args) -> obj.instanceMethod(args) Class::instanceMethod
    Constructor args -> new Class(args) Class::new

    範例:

    List<String> names = Arrays.asList("Alice", "Bob", "Charlie");

// Lambda
names.forEach(s -> System.out.println(s));

// Method Reference
names.forEach(System.out::println);

// Lambda (排序)
names.sort((s1, s2) -> s1.compareToIgnoreCase(s2));

// Method Reference (排序)
names.sort(String::compareToIgnoreCase);

    4. 總結¶

    • **巢狀類別**用於邏輯分組與封裝，分為 Static, Member, Local, Anonymous 四種。
    • **Lambda 表達式**簡化了 Functional Interface 的實作，使程式碼更簡潔、易讀。
    • Functional Interfaces (如 Predicate, Consumer, Function) 是 Lambda 的基礎。
    • Method References 提供了更進一步的簡化語法。

    5. 練習建議¶

    1. 建立一個 Student 類別，並使用 Comparator 與 Lambda 表達式依據成績排序。
    2. 使用 Predicate 過濾一個整數 List，只保留偶數。
    3. 使用 Function 將一個字串 List 全部轉換為大寫。
