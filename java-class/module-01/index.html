    Module 01: Java 簡介 (Introduction to Java)¶

    Java SE 21 程式語言設計

    學習目標¶

    完成本章節後，您將能夠： - 了解 Java 語言的起源、版本演進與主要特性 - 區分 JVM, JRE, JDK 的功能與差異 - 完成 Java 開發環境的設定 - 撰寫並執行第一個 Java 程式 (Hello World) - 理解物件導向 (Object-Oriented) 的基本概念 (類別、物件、繼承)

    1. 什麼是 Java？¶

    Java 是由 Sun Microsystems（現為 Oracle）於 1995 年開發的物件導向程式語言。Java 的核心設計理念是「一次撰寫，到處執行」（Write Once, Run Anywhere, WORA），這使得 Java 程式具有極高的跨平台特性。

    Java 的主要特色¶

    1. 跨平台性 (Platform Independent)：Java 原始碼編譯後產生 Bytecode (位元組碼)，可在任何安裝了 JVM 的平台上執行。
    2. 物件導向 (Object-Oriented)：Java 是純物件導向語言，支援封裝、繼承、多型等特性。
    3. 自動記憶體管理：透過垃圾回收機制 (Garbage Collection, GC) 自動管理記憶體，減少記憶體洩漏風險。
    4. 安全性：Java 提供沙箱機制與強型別檢查，確保程式執行的安全性。
    5. 豐富的 API：擁有龐大的標準類別庫，涵蓋網路、IO、並發、資料庫等領域。

    Java 版本¶

    • Java SE (Standard Edition)：標準版，Java 的核心，也是本課程的重點。
    • Java EE (Jakarta EE)：企業版，用於開發大型企業應用程式。
    • Java ME (Micro Edition)：微型版，用於嵌入式裝置。
    • LTS (Long Term Support)：長期支援版本（如 Java 8, 11, 17, 21），適合企業長期使用。本課程使用最新的 Java 21。

    2. Java 運作原理 (JVM, JRE, JDK)¶

    JVM (Java Virtual Machine)¶

    • Java 虛擬機器：負責執行 Java Bytecode。
    • 跨平台核心：不同作業系統 (Windows, Linux, macOS) 有對應的 JVM 實作，但它們都能執行相同的 Bytecode。

    JRE (Java Runtime Environment)¶

    • Java 執行環境：包含 JVM + Java 標準類別庫。
    • 用途：僅需執行 Java 程式的使用者安裝 JRE 即可。

    JDK (Java Development Kit)¶

    • Java 開發工具包：包含 JRE + 開發工具 (編譯器 javac, 除錯器 jdb 等)。
    • 用途：開發者必須安裝 JDK。

    3. 第一個 Java 程式¶

    程式碼範例：HelloWorld.java¶

    這是最簡單的 Java 程式，用於輸出一段文字。

    HelloWorld.java
    1
2
3
4
5
    public class HelloWorld {
    public static void main(String[] args){
        System.out.println("Hello Java");
    }
}

    程式碼解析¶

    1. public class HelloWorld：
      • 定義一個公開的類別 (Class) 名為 HelloWorld。
      • 注意：在 Java 中，若類別是 public 的，檔名必須與類別名稱完全相同 (大小寫敏感)，即 HelloWorld.java。
    2. public static void main(String[] args)：
      • 這是 Java 程式的 進入點 (Entry Point)。當程式執行時，JVM 會尋找並執行這個方法。
      • public：表示此方法可被外部 (JVM) 呼叫。
      • static：表示此方法屬於類別本身，不需要建立物件即可執行。
      • void：表示此方法不回傳任何值。
      • String[] args：接收命令列參數 (Command Line Arguments)。
    3. System.out.println("Hello Java");：
      • 將字串 "Hello Java" 輸出到控制台 (Console) 並換行。

    編譯與執行¶

    1. 編譯 (Compile)：將原始碼 (.java) 轉換為位元組碼 (.class)。

      Bash
      1
      javac HelloWorld.java
      成功後會產生 HelloWorld.class 檔案。

    2. 執行 (Run)：啟動 JVM 執行位元組碼。

      Bash
      1
      java HelloWorld
      注意：執行時使用的是 類別名稱，不需要加上 .class 副檔名。

    4. 物件導向概念 (Object-Oriented Concepts)¶

    Java 程式設計的核心是物件導向。我們透過定義 類別 (Class) 來描述事物的特徵，並透過建立 物件 (Object) 來使用這些功能。

    核心概念¶

    • 類別 (Class)：物件的藍圖或模板。定義了物件擁有的 屬性 (Fields/Attributes) 和 行為 (Methods)。
      • 例如：「狗」這個概念，有名字 (屬性)，會接球 (行為)。
    • 物件 (Object)：類別的實例 (Instance)。是記憶體中實際存在的個體。
      • 例如：「你家養的那隻叫 Rex 的狗」。

    範例情境¶

    我們模擬一個情境：一個人 (Person) 和一隻狗 (Dog) 玩球 (Ball)。

    這是一個父類別，代表所有動物的通用特徵。

    狗是一種動物，所以繼承自 Animal。

    public class Dog extends Animal {
    String name; // 屬性：名字

    // 行為：接球
    void fetch(Ball ball) {
        System.out.println(name + " is fetching the ball...");
        ball.find(); // 呼叫球的方法
        ball.chew();
    }
}
    - extends Animal：表示繼承。Dog 擁有 Animal 的所有特徵。 - fetch(Ball ball)：定義了一個方法，接收一個 Ball 物件作為參數。

    人負責發起遊戲。

    public class Person {
    void play() {
        // 1. 建立 Dog 物件 (實例化)
        Dog dog = new Dog();
        dog.name = "Rex"; // 設定屬性

        // 2. 建立 Ball 物件
        Ball ball = new Ball();

        // 3. 呼叫方法 (互動)
        dog.fetch(ball);
    }
}
    - new Dog()：使用 new 關鍵字建立一個 Dog 類別的新實例 (物件)。 - dog.name = "Rex"：存取並設定物件的屬性。 - dog.fetch(ball)：呼叫物件的方法，並將另一個物件 (ball) 傳遞進去。

    5. 總結¶

    • Java 是一種跨平台、物件導向的強型別語言。
    • 開發 Java 需要 JDK，執行 Java 需要 JRE。
    • Java 程式執行流程：.java (原始碼) -> javac (編譯) -> .class (Bytecode) -> java (JVM 執行)。
    • 類別是藍圖，物件是實例。透過方法呼叫讓物件之間進行互動。

    6. 練習建議¶

    1. 確認已安裝 JDK 21 並設定好環境變數 (JAVA_HOME, PATH)。
    2. 使用文字編輯器撰寫 HelloWorld.java 並在終端機編譯執行。
    3. 嘗試修改 HelloWorld.java，輸出你的名字。
    4. 建立 Dog, Ball, Person 類別，並撰寫一個包含 main 方法的測試類別來執行 Person 的 play 方法。
